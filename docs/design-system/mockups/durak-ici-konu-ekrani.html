async/await Gerçekte Ne Yapar?

Süre 18 dkTip MekanizmaÖnkoşul Thread temelleri ✓
KANCA
await yazdığında thread beklemiyor. Peki 30 saniyelik HTTP çağrısı sırasında o thread nerede?

Bu sorunun cevabını bilmeden yazılan her async kod, çalışan ama anlaşılmamış koddur. 18 dakika sonra cevabı kendi cümlenle verebileceksin.

KAVRAM — ZİHİNSEL MODEL

Restorandaki garsonu düşün. Siparişini mutfağa iletir ve masanın başında beklemez — diğer masalara bakar. Yemek hazır olunca mutfak ona haber verir, kaldığı yerden devam eder.

await tam olarak budur: thread (garson), I/O işini (mutfak) başlatır ve thread pool'a geri döner. İş bittiğinde bir continuation tetiklenir ve metot kaldığı satırdan devam eder. Bekleyen bir thread yok; bekleyen sadece bir "devam noktası" kaydı var.

KOD — DERLEYİCİNİN GÖRDÜĞÜ
OrderService.csKopyala ⧉
public async Task<Order> GetOrderAsync(int id)
{
    // buraya kadar çağıran thread'desin
    var order = await _http.GetFromJsonAsync<Order>($"/orders/{id}");
    // bu satır, belki de BAŞKA bir thread'de çalışacak
    return order;
}
Vurgulu satır: Derleyici bu metodu bir state machine'e çevirir. await noktası, "buradan devam et" etiketidir — metodun geri kalanı continuation olarak paketlenir.
YAYGIN YANILGI
⚠️

"async = çoklu thread" değildir

async/await bir eşzamanlılık (concurrency) aracıdır, paralellik aracı değil. Tek thread'li bir ortamda bile (örn. eski ASP.NET SynchronizationContext) çalışır. CPU-bound iş için await değil, Task.Run düşünülür — ve web API'de o da çoğu zaman yanlıştır.

await sırasında HTTP çağrısı 30 saniye sürerse, isteği başlatan thread bu sürede ne yapar?

[a]Cevap gelene kadar bloklanır ve bekler
[b]Thread pool'a döner, başka istekleri işler
[c]Yeni bir thread yaratıp işi ona devreder
Doğru. Thread serbest kalır — bu yüzden async, az thread'le çok isteği ayakta tutar. Cevap geldiğinde continuation, pool'daki uygun bir thread'de koşar.
PRODUCTION NOTU

Sahada nerede patlar?

Klasik olay: birisi .Result veya .Wait() ile async'i senkrona çevirir. Yük altında thread pool açlığı (starvation) başlar, API yanıt veremez olur — CPU %10'dayken sistem "donar". Log'da göreceğin şey: artan istek kuyruğu, sabit düşük CPU.

ÖZET + SONRAKİ DURAK

Bu duraktan cebinde kalanlar

  • await beklemez; devam noktası kaydeder, thread'i serbest bırakır.
  • Derleyici async metodu state machine'e çevirir.
  • async ≠ paralellik; eşzamanlılık aracıdır.
  • .Result / .Wait() production'da thread pool açlığının bir numaralı sebebidir.
